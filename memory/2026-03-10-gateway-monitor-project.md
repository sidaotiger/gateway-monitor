## 一、需求文档

### 1.1 背景与目标

**背景：**
OpenClaw Gateway 是核心服务，需要 7×24 小时运行。目前没有自动监控和恢复机制，Gateway 挂了需要人工发现和重启。

**目标：**
实现 Gateway 自动化监控与恢复，降低人工干预成本。

### 1.2 功能需求

| 需求ID | 需求描述 | 优先级 |
|--------|----------|--------|
| F1 | 监控 Gateway 端口 18789 是否可达 | P0 |
| F2 | 检测到挂了自动执行 `openclaw gateway start` 重启 | P0 |
| F3 | 状态变化时发送飞书通知 | P0 |
| F4 | 每隔 5 分钟检查一次 | P1 |
| F5 | 本地日志记录 | P1 |

### 1.3 非功能需求

| 需求ID | 需求描述 | 优先级 |
|--------|----------|--------|
| NF1 | 状态变化才通知，避免重复刷屏 | P0 |
| NF2 | 脚本稳定运行，异常不crash | P0 |

---

## 二、技术方案

### 2.1 实现方式
- 语言：Python 3
- 监控方式：TCP 端口连接检测
- 通知：飞书自定义机器人 Webhook

### 2.2 核心逻辑

```
while True:
    检查端口 18789
    if 可连接:
        记录存活
    else:
        if 之前是存活状态:
            尝试重启 Gateway
            发送飞书通知
    等待 5 分钟
```
